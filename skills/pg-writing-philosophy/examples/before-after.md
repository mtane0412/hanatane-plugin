# Before/After 実例集

このドキュメントは、ポール・グレアム風の編集の実例を示します。各例では、元の文章(Before)と編集後の文章(After)を比較し、変更理由を説明します。

---

## 例1: 堅苦しい論文調を会話調に

### Before

```
本稿においては、スタートアップ企業における成長戦略の重要性について論じることとする。一般的に、スタートアップ企業が成功を収めるためには、持続的な成長を実現することが不可欠であると考えられている。しかしながら、多くのスタートアップ企業が直面する課題は、いかにして適切な成長戦略を立案し、実行するかという点にある。本稿では、この課題に対する解決策を提示することを目的とする。
```

### After

```
スタートアップにとって成長は重要だ。しかし、どう成長すればいいのだろうか?

多くの創業者がこの問いに苦しんでいる。適切な成長戦略を見つけるのは難しい。この記事では、その解決策を示したい。
```

### 変更理由

1. **「本稿」「である」調 → 「だ」調**: 会話調に変換
2. **長い文を分割**: 1文1アイデアに
3. **疑問文を追加**: 読者を巻き込む
4. **冗長な表現を削除**: 「〜することとする」「〜と考えられている」などを削除
5. **段落を分割**: 視覚的な休憩ポイントを作成

## 例2: 抽象的すぎる表現を具体化

### Before

```
多くの企業が失敗する主な理由は、市場のニーズを適切に把握できないことにある。顧客が本当に求めているものを理解せずにプロダクトを開発してしまうケースが散見される。
```

### After

```
なぜスタートアップの90%は失敗するのか? 最大の理由は、誰も欲しがらないものを作るからだ。

例えば、ある企業は「革新的なSNS」を作った。しかし、誰も使わなかった。なぜか? ユーザーはすでにTwitterとFacebookで満足していたからだ。

この企業は、実際にユーザーに話を聞かなかった。自分たちが「革新的」だと思うものを作っただけだった。
```

### 変更理由

1. **「多くの」→ 「90%」**: 具体的な数字を使用
2. **疑問文で開始**: 読者の注意を引く
3. **具体例を追加**: 抽象的な主張に具体例を添える
4. **短い文に分割**: リズムを作る
5. **会話調**: 「〜にある」→「〜からだ」

## 例3: 専門用語を避ける

### Before

```
RESTful APIのエンドポイント設計において、リソース指向アーキテクチャを採用することは、スケーラビリティとメンテナビリティの観点から推奨される。HTTPメソッドを適切に使用し、ステートレスな設計を心がけることで、システムの疎結合性を高めることができる。
```

### After

```
Web APIを設計する時、何に注意すべきか?

まず、URLをシンプルにしよう。例えば、ユーザー情報を取得するなら `/users/123` のようにする。複雑にしない。

次に、HTTPの標準的な方法を使おう。データを取得するなら GET、作成するなら POST だ。

最後に、サーバーは前回の状態を覚えないようにしよう。これで、システムがシンプルになる。
```

### 変更理由

1. **専門用語を削除または説明**: RESTful、リソース指向、スケーラビリティなど
2. **具体例を追加**: `/users/123` のような実例
3. **疑問文で開始**: トピックを明確にする
4. **短い段落に分割**: 各ポイントを独立させる
5. **簡単な言葉**: 「疎結合性」→「シンプル」

## 例4: 受動態を能動態に

### Before

```
この問題は多くの研究者によって研究されている。様々なアプローチが試みられてきたが、決定的な解決策は見出されていない。今後、さらなる検討が必要とされる。
```

### After

```
多くの研究者がこの問題を研究している。彼らは様々なアプローチを試した。しかし、誰も決定的な解決策を見つけていない。

私たちはさらに考える必要がある。
```

### 変更理由

1. **受動態 → 能動態**: 「研究されている」→「研究している」
2. **「見出されていない」→「見つけていない」**: より直接的な表現
3. **「必要とされる」→「必要がある」**: シンプルに
4. **主語を明確に**: 「私たち」を追加
5. **段落を分割**: 最後の文を強調

## 例5: 冗長な表現を削除

### Before

```
基本的に、スタートアップが成功することができるかどうかは、プロダクトの質に関して大きく依存すると考えることができる。言い換えれば、良いプロダクトを作ることができれば、成功する可能性が高いと言える。
```

### After

```
スタートアップの成功は、プロダクトの質で決まる。良いプロダクトがあれば、成功する可能性が高い。
```

### 変更理由

1. **「基本的に」削除**: 不要な修飾語
2. **「〜することができる」→「〜する」**: シンプルに
3. **「〜に関して」削除**: 冗長
4. **「〜と考えることができる」削除**: 当たり前
5. **「言い換えれば」削除**: 繰り返しを避ける
6. **40語 → 20語**: 半分の長さで同じ意味

## 例6: 段落を短く分割

### Before

```
スタートアップを成功させるには、いくつかの重要な要素がある。まず、優れたプロダクトが必要だ。次に、適切なチームが必要だ。そして、十分な資金も必要だ。しかし、これらすべてよりも重要なものがある。それは、諦めない姿勢だ。多くのスタートアップが失敗する最大の理由は、諦めてしまうことだ。困難に直面した時、多くの人は諦める。しかし、成功する人は続ける。
```

### After

```
スタートアップを成功させるには何が必要か?

優れたプロダクトだ。

適切なチームだ。

十分な資金だ。

しかし、これらすべてよりも重要なものがある。

諦めない姿勢だ。

多くのスタートアップが失敗する最大の理由は、諦めてしまうことだ。困難に直面した時、ほとんどの人は諦める。

しかし、成功する人は続ける。
```

### 変更理由

1. **疑問文で開始**: 読者を引き込む
2. **各要素を独立した段落に**: 視覚的インパクト
3. **短い文で強調**: 「諦めない姿勢だ」を独立させる
4. **リズムを作る**: 同じ構造の繰り返し
5. **最後の文を独立**: パンチラインを強調

## 例7: 具体例を追加

### Before

```
良いアイデアは、大きな問題から生まれるとは限らない。実は、小さな問題から始まることが多い。自分が直面している問題を解決しようとすることが、良いスタートアップのアイデアにつながる。
```

### After

```
良いアイデアは、大きな問題から生まれるとは限らない。実は、小さな問題から始まることが多い。

例えば、Airbnbはどう始まったか? 創業者たちは家賃が払えなかった。そこで、部屋の一部を貸し出すことを思いついた。小さな問題だ。

Stripeも同様だ。創業者たちは、オンライン決済があまりにも複雑だと感じた。自分たちのために、シンプルな決済システムを作った。

つまり、自分が直面している問題を解決しようとすることが、良いスタートアップのアイデアにつながる。
```

### 変更理由

1. **具体例を2つ追加**: Airbnb と Stripe
2. **疑問文を使用**: 「どう始まったか?」
3. **ストーリーテリング**: 実際のエピソードを語る
4. **「つまり」で一般化**: 例から教訓を導く
5. **段落を分割**: 各例を独立させる

## 例8: 驚きの要素を追加

### Before

```
スタートアップは、速く動くことが重要だ。市場の変化に迅速に対応する必要がある。競合に負けないためにも、スピードが鍵となる。
```

### After

```
多くの人は、スタートアップは速く動くべきだと考えている。

これは間違いだ。

速く動くことよりも、正しい方向に動くことの方が重要だ。間違った方向に速く進んでも、崖に速く到達するだけだ。

もちろん、スピードは重要だ。しかし、方向が間違っていたら意味がない。

まず正しい方向を見つけよう。その後、速く進めばいい。
```

### 変更理由

1. **常識に挑戦**: 「速さが重要」という常識に反論
2. **驚きの主張**: 「これは間違いだ」で読者を驚かせる
3. **比喩を使用**: 「崖に速く到達する」
4. **反論を認める**: 「もちろん、スピードは重要だ」
5. **行動を促す**: 「まず〜しよう」

## 編集プロセスの例

最後に、1つの段落が複数回の編集を経てどう変化するかを示します。

### 第1稿(初稿)

```
スタートアップにおいて成功を収めるためには、多様な要素が複雑に絡み合っており、それらの要素を適切にマネジメントすることが求められる。特に、プロダクト開発、マーケティング活動、資金調達、チームビルディングなどの各領域において、バランスの取れた戦略を立案し、実行していくことが重要であると考えられる。
```

### 第2稿(冗長な表現を削除)

```
スタートアップが成功するには、多くの要素をマネジメントする必要がある。プロダクト開発、マーケティング、資金調達、チームビルディングなど、各領域でバランスの取れた戦略を実行することが重要だ。
```

### 第3稿(会話調に変換)

```
スタートアップが成功するには何が必要か? 多くのことをマネジメントする必要がある。プロダクト開発、マーケティング、資金調達、チーム作り。すべてをバランスよくやらなければならない。
```

### 第4稿(短く分割)

```
スタートアップが成功するには何が必要か?

多くのことをやらなければならない。プロダクト開発。マーケティング。資金調達。チーム作り。

すべてをバランスよくやる必要がある。
```

### 最終稿(驚きの要素を追加)

```
スタートアップが成功するには何が必要か?

多くの人は、すべてをバランスよくやることだと考えている。プロダクト開発、マーケティング、資金調達、チーム作り。

これは間違いだ。

実際は、1つのことに集中すべきだ。プロダクトだ。優れたプロダクトがあれば、他のすべてはついてくる。
```

## まとめ: 編集のチェックリスト

- [ ] 堅苦しい表現 → 会話調
- [ ] 長い文 → 短い文
- [ ] 抽象的 → 具体的
- [ ] 専門用語 → 簡単な言葉
- [ ] 受動態 → 能動態
- [ ] 冗長な表現を削除
- [ ] 段落を短く分割
- [ ] 具体例を追加
- [ ] 驚きの要素を追加
- [ ] 疑問文を使用

これらの変更を加えることで、PG風の明快で魅力的な文章になります。
