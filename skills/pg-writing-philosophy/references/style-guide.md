# 文体とトーンのガイド

このドキュメントは、ポール・グレアム風の文体とトーンを実現するための具体的なガイドラインを提供します。

## 文体の基本原則

### 0. saltintesta: 透明な文章を目指す

ポール・グレアムが目指す理想の文章は「saltintesta」です。これはイタリア語で「頭に飛び込む」という意味です。

**saltintestaとは**:
- アイデアが頭に直接飛び込み、言葉がそこにあったことに気づかない状態
- 読者が文章の存在を意識せず、アイデアだけが残る
- 透明なガラスのような文章

**達成方法**:
- シンプルな言葉を使う
- 複雑な文法を避ける
- 読みにくさを排除する

読みにくさは読者のエネルギーを消費し、アイデアへの集中を妨げます。派手な文章は誠実さの欠如を隠すためにも使われます。シンプルに書けば、言うことがなければそれが明らかになります。

### 1. 短い文を使う

ポール・グレアムの文は短いです。平均的に15-20語程度です。

**目安**:
- 短い文: 10語以下
- 標準的な文: 15-20語
- 長い文: 25語以上(避けるべき)

**悪い例(40語)**:
```
スタートアップを成功させるためには、優れたプロダクトを作り、ユーザーを理解し、素早く改善していくことが重要であり、そのためにはチーム全体が同じ目標に向かって協力する必要がある。
```

**良い例(各文10-15語)**:
```
スタートアップを成功させるには何が必要か? まず優れたプロダクトだ。次にユーザーの理解だ。そして素早い改善だ。これにはチームの協力が不可欠だ。
```

### 2. 短い段落を使う

段落も短くしてください。1段落は3-5文が理想です。

**段落の長さの目安**:
- 1行: 強調したいポイント
- 2-3文: 標準的な段落
- 4-5文: やや長い段落
- 6文以上: 長すぎる(分割を検討)

**悪い例(1段落に8文)**:
```
スタートアップは難しい。多くの人が失敗する。なぜだろうか? 理由はいくつかある。まずプロダクトが悪い場合だ。次に市場がない場合だ。また資金が尽きることもある。しかし最大の理由は、諦めることだ。
```

**良い例(3段落に分割)**:
```
スタートアップは難しい。多くの人が失敗する。なぜだろうか?

理由はいくつかある。プロダクトが悪い。市場がない。資金が尽きる。

しかし最大の理由は、諦めることだ。
```

### 3. 単純な語彙を使う: ゲルマン語由来の言葉を優先する

難しい言葉よりも、誰でも知っている簡単な言葉を使ってください。

**言葉の選択原則**:
英語では、ラテン語由来の堅い言葉ではなく、ゲルマン語由来のシンプルな言葉を選びます。日本語でも同様に、和語や日常語を漢語や外来語より優先します。

**ラテン語由来 → ゲルマン語由来(英語)**:
- utilize → use
- facilitate → help
- commence → start
- terminate → end
- purchase → buy

**漢語・形式語 → 和語・日常語(日本語)**:
- 利用する → 使う
- 実施する → する、やる
- 検討する → 考える
- 把握する → わかる、知る
- 保有する → 持つ
- 発生する → 起きる
- 認識する → 気づく、わかる
- 判断する → 決める
- 獲得する → 得る、手に入れる

**例**:
```
悪い: 我々は新しい戦略を検討し、実施することを決定した
良い: 私たちは新しい戦略を考え、やることに決めた
```

**原則**: 小学生が理解できる言葉を使う

## トーンの原則

### 1. カジュアルだが誠実

友人と話すようにカジュアルに書いてください。しかし、軽薄にならないように注意してください。誠実さを保ってください。

**バランス**:
- 堅苦しすぎる ← **適切** → 軽薄すぎる

**堅苦しすぎる例**:
```
本稿において論述する内容は、スタートアップの成長戦略に関する考察である。
```

**適切な例**:
```
スタートアップの成長戦略について考えてみよう。
```

**軽薄すぎる例**:
```
スタートアップの成長戦略ってヤバいよね!笑
```

### 2. 自信を持つが傲慢にならない

自分の意見を自信を持って述べてください。しかし、傲慢にならないように注意してください。

**良いバランス**:
```
私はこう考えている: スタートアップは小さく始めるべきだ。もちろん、これに反対する人もいるだろう。しかし私の経験では、小さく始めた方がうまくいくことが多い。
```

**傲慢な例**(避けるべき):
```
スタートアップは小さく始めるべきだ。これに反対する人は間違っている。私が正しい。
```

**自信がなさすぎる例**(避けるべき):
```
私の考えでは、もしかしたら、スタートアップは小さく始めた方がいいかもしれません。ただの個人的な意見ですが...
```

### 3. 読者を尊重する

読者を見下さないでください。読者が理解できないことを前提にしないでください。

**良い例**:
```
APIとは何か? 簡単に言えば、プログラム同士が通信する方法だ。
```

**悪い例**(見下している):
```
APIという言葉を知らない人もいるかもしれないので、初歩から説明しよう。
```

## 文のパターン

### 1. 疑問文で始める

段落や新しいトピックを疑問文で始めるのは効果的です。

**パターン**:
```
[疑問文]? [答え]。[詳細な説明]。
```

**例**:
```
なぜスタートアップは失敗するのか? 最大の理由は、誰も欲しがらないものを作るからだ。これは驚くべきことに、最も一般的な失敗の原因だ。
```

### 2. 短い答えで強調する

疑問文の後に、非常に短い答えを置くと効果的です。

**パターン**:
```
[疑問文]? [1-3語の答え]。
```

**例**:
```
スタートアップに必要なものは何か? 成長だ。

良いアイデアの条件とは? 驚きだ。

プログラマーが最も重視すべきものは? シンプルさだ。
```

### 3. 対比を使う

対比は、ポイントを明確にする優れた方法です。

**パターン**:
```
[一般的な考え]。しかし実際は[真実]。
```

**例**:
```
多くの人は、スタートアップには大きなビジョンが必要だと考えている。しかし実際は、小さな問題を解決することから始まることが多い。
```

### 4. 具体例で始める

抽象的な主張の前に、具体例を置くことも効果的です。

**パターン**:
```
[具体例]。これは[抽象的な原則]を示している。
```

**例**:
```
Airbnbは家賃を払えないことから始まった。Stripeはオンライン決済が複雑すぎることから始まった。これは、優れたプロダクトは小さな問題から始まることを示している。
```

## 避けるべき表現

### 1. 冗長な表現

以下の表現は削除してください:

| 冗長な表現 | 簡潔な表現 |
|----------|----------|
| ～することができる | ～できる |
| ～と言える | (削除) |
| ～と考えられる | ～と思う |
| ～の場合 | ～なら |
| ～に関して | ～について |
| ～という事実 | (削除) |
| ～ということ | (削除) |
| 基本的に | (削除) |
| 実際のところ | 実際は |

**例**:
```
悪い: 基本的に、スタートアップが成功することができるかどうかは、プロダクトの質に関して依存すると考えられる
良い: スタートアップの成功は、プロダクトの質で決まる
```

### 2. 受動態

可能な限り能動態を使ってください。

**受動態 → 能動態**:
```
受動: この問題は解決された
能動: 私たちはこの問題を解決した

受動: 多くの企業によって採用されている
能動: 多くの企業が採用している
```

### 3. 名詞化

動詞を名詞化すると、文章が固くなります。動詞のまま使ってください。

**名詞化 → 動詞**:
| 名詞化 | 動詞 |
|-------|------|
| 実施する | する、やる |
| 検討する | 考える |
| 提供する | 提供する(そのまま) |
| 利用する | 使う |
| 確認する | 確かめる、見る |

**例**:
```
悪い: プロダクトの改善を実施する
良い: プロダクトを改善する
```

### 4. 曖昧な数量詞

「多くの」「いくつかの」「最近」などの曖昧な表現は避け、具体的な数字や時期を使ってください。

**曖昧 → 具体的**:
```
曖昧: 多くのスタートアップが失敗する
具体的: 90%のスタートアップが失敗する

曖昧: 最近のトレンド
具体的: 2023年のトレンド

曖昧: いくつかの理由
具体的: 3つの理由
```

## 修飾語(Qualification)の使い方

### 修飾語は弱さではなく、精密さのツールである

多くの人は、「おそらく」「かもしれない」などの修飾語を弱さだと考えます。しかしポール・グレアムは、修飾語は確信の度合いを正確に表現する重要なスキルだと主張します。

### 修飾語の原則

**確信の度合いを正確に表現する**:
- 確信していることは修飾なしで断言できる
- 疑わしい点は「perhaps」「かもしれない」で距離を置く
- 中間的な確信度には適切な修飾語を使う

**例**:
```
100%確信: スタートアップには成長が重要だ
80%確信: スタートアップには成長が重要だと思う
50%確信: スタートアップには成長が重要かもしれない
```

### 修飾語の多様性

修飾語はスカラー(0%〜100%)ではありません。50種類以上の意味を持ちうります:

1. **適用範囲の修飾**: 「ほとんどの場合」「多くの状況で」
2. **知識の根拠の修飾**: 「私の経験では」「データによれば」
3. **感情の修飾**: 「残念ながら」「驚くべきことに」
4. **反証可能性の修飾**: 「現時点では」「証拠がない限り」

**例**:
```
適用範囲: ほとんどのスタートアップは失敗する(すべてではない)
知識の根拠: 私の経験では、小さく始めた方がいい
感情: 残念ながら、多くの創業者がこれを知らない
反証可能性: 現時点では、AIが人間を超える証拠はない
```

### 過度な修飾を避ける

修飾語は重要ですが、過度に使うと文章が弱くなります:

**悪い例(過度な修飾)**:
```
おそらく、もしかしたら、スタートアップには成長が重要かもしれないと言えるかもしれない。
```

**良い例(適切な修飾)**:
```
スタートアップには成長が重要だと思う。
```

## 声に出して読む

最も重要なテストは、声に出して読むことです。

**チェックポイント**:
1. つかえずに読めるか?
2. 自然に聞こえるか?
3. 友人との会話のように聞こえるか?
4. 呼吸が苦しくならないか?(文が長すぎる場合、呼吸が苦しくなる)

声に出して読んで不自然な部分は、書き直してください。

## リズムとテンポ

### 1. 文の長さを変化させる

すべての文が同じ長さだと、単調になります。長い文と短い文を混ぜてください。

**例**:
```
スタートアップは難しい。(短)
多くの人が失敗する理由は、プロダクトが悪いからだ。(中)
しかし最大の理由は諦めることだ。(中)
諦めるな。(極短)
```

### 2. パンチラインを短くする

最も重要なポイント(パンチライン)は、短い文で表現してください。

**例**:
```
スタートアップを成功させるには、優れたプロダクトが必要だ。ユーザーを理解する必要がある。素早く改善する必要がある。しかし最も重要なのは何か?

諦めないことだ。
```

### 3. 繰り返しを使う

同じ構造を繰り返すと、リズムが生まれます。

**例**:
```
良いプロダクトは小さく始まる。
良いプロダクトはシンプルだ。
良いプロダクトは実際の問題を解決する。
```

## まとめ: PG風文体のチェックリスト

文章を書いた後、以下をチェックしてください:

- [ ] 文は短いか?(平均15-20語)
- [ ] 段落は短いか?(3-5文)
- [ ] 簡単な言葉を使っているか?
- [ ] 会話調になっているか?
- [ ] 疑問文を使っているか?
- [ ] 具体例があるか?
- [ ] 冗長な表現を削除したか?
- [ ] 受動態を能動態に変換したか?
- [ ] 声に出して読んで自然か?
- [ ] リズムとテンポがあるか?

これらすべてがYesなら、PG風の文体になっています。
