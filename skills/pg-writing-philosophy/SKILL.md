# pg-writing-philosophy

このスキルは、ユーザーがポール・グレアム風のエッセイを執筆する際に使用されます。ユーザーが「PG風に書く」「ポール・グレアムのように」「明快な文章にしたい」「エッセイを書く」「会話調の文体で」などのフレーズを使った時にこのスキルを読み込んでください。

---

## ポール・グレアムの執筆哲学

ポール・グレアムは、明快さとシンプルさを最重視する執筆スタイルで知られています。彼のエッセイは、複雑なアイデアを誰にでも理解できる形で表現し、読者に直接話しかけるような親しみやすさを持っています。

### 1. 明快さを最優先する

あなたが書く文章は、明快でなければなりません。複雑な言葉、専門用語、回りくどい表現を避けてください。読者がすぐに理解できる言葉を選んでください。

**悪い例**:
```
当該システムにおける実装上の課題は、複数のステークホルダー間における認識の齟齬に起因する。
```

**良い例**:
```
このシステムの問題は、関係者の間で認識がずれていることだ。
```

文章は短く、直接的にしてください。1つの文で1つのアイデアを伝えてください。長い文は分割してください。

### 2. 会話調で書く

読者に話しかけるように書いてください。堅苦しい書き言葉ではなく、友人に説明するような口調を使ってください。

- 「である」調よりも「だ」「です」調を使ってください
- 二人称(あなた、君)を積極的に使ってください
- 疑問文を使って読者を巻き込んでください
- 時には省略形や口語表現を使ってください

**ポイント**: ポール・グレアムは、エッセイを「話すように書く」と表現しています。声に出して読んだ時に自然に聞こえるかどうかを確認してください。

### 3. 具体例で説明する

抽象的な概念は、必ず具体例で説明してください。読者は抽象論だけでは理解できません。

**悪い例**:
```
スタートアップは、従来の企業とは異なる成長曲線を描く。
```

**良い例**:
```
スタートアップは、従来の企業とは異なる成長曲線を描く。例えば、Airbnbは最初の3年間ほとんど成長しなかったが、その後急激に成長した。
```

実際の会社名、人名、具体的な数字、エピソードを使ってください。読者が頭の中でイメージできるようにしてください。

### 4. リライトを重視する

最初の下書きは捨てる覚悟で書いてください。良いエッセイは、何度も書き直すことで生まれます。

**リライトのプロセス**:
1. まず思いつくままに書く
2. 読み返して、わかりにくい部分を見つける
3. その部分を書き直す
4. 全体を読み返して、流れを確認する
5. さらに簡潔にできる部分を削る
6. これを何度も繰り返す

削除を恐れないでください。良い文章は、削ることで生まれます。不要な単語、不要な文、不要な段落を積極的に削除してください。

### 5. アイデアから始める

良いエッセイは、良いアイデアから始まります。書き始める前に、何を伝えたいのかを明確にしてください。

**問いかけ**:
- このエッセイで伝えたい核心的なアイデアは何か?
- 読者に何を気づいてほしいか?
- なぜこのトピックが重要なのか?

アイデアが明確でない場合は、書き始めないでください。まずアイデアを明確にすることに時間を使ってください。

### 6. 驚きの要素を入れる

読者が「へえ、そうなんだ」と思う瞬間を作ってください。予想外の視点、意外な事実、新しい気づきを提供してください。

常識に反する主張をする時は、なぜそれが正しいのかを丁寧に説明してください。ただし、驚かせるために嘘をついてはいけません。

### 7. 短い文、短い段落

文は短くしてください。段落も短くしてください。読者は長い文章を読むのに疲れます。

**目安**:
- 1文: 平均15-20語以内
- 1段落: 3-5文以内

長い段落は、読者を疲れさせます。適切な位置で段落を分割してください。

### 8. 誠実であれ

自分が本当に信じていることを書いてください。人気を得るため、誰かを喜ばせるために嘘をついてはいけません。

不確実なことは不確実だと認めてください。わからないことは、わからないと言ってください。誠実さは、読者の信頼を得る最良の方法です。

## 執筆プロセス

あなたがユーザーの文章を編集する際は、以下のプロセスに従ってください:

1. **全体を読む**: まず文章全体を読んで、核心的なアイデアを理解してください
2. **構造を確認する**: 論理の流れが明確かどうかを確認してください
3. **明快さをチェック**: わかりにくい表現、専門用語、回りくどい文を見つけてください
4. **会話調に変換**: 堅苦しい表現を会話調に変換してください
5. **具体例を追加**: 抽象的な部分に具体例を追加してください
6. **削除する**: 不要な言葉、文、段落を削除してください
7. **短くする**: 長い文、長い段落を分割してください
8. **読み返す**: 最終的に声に出して読んで、自然に聞こえるか確認してください

## 使用例

### ケース1: 下書きの編集
ユーザーが「この下書きをPG風に編集して」と言った場合:
1. このスキルを読み込む
2. 文章全体を読んで理解する
3. 上記の原則に従って編集する
4. 変更理由を説明する

### ケース2: 新規執筆支援
ユーザーが「○○についてエッセイを書きたい」と言った場合:
1. このスキルを読み込む
2. 核心的なアイデアを明確にする質問をする
3. アイデアが明確になったら、執筆をサポートする
4. PGの原則に従った構造を提案する

### ケース3: 反復的な改善
ユーザーが「さらに改善できる?」と言った場合:
1. 現在の文章を再度読む
2. さらに明快にできる部分を見つける
3. さらに削除できる部分を見つける
4. 改善案を提示する

---

**重要**: このスキルは、単に文章を書き換えるだけではありません。ユーザーが明快で、シンプルで、誠実なエッセイを書けるように支援することが目的です。機械的な変換ではなく、アイデアの本質を引き出すことに焦点を当ててください。
